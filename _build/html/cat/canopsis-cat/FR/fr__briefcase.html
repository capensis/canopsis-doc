Contents¶

Description¶

In Canopsis, the briefcase allows to associate one or more files to an entity.

All associated documents are located in a file storage.

Frontend¶

Briefcase component¶

The frontend MUST provide a component to explore associated files and folders, with the following layouts :

  • list view : each file is listed with its full path
  • tree view : each folder is expandable/collapsible
  • icon view : each file and folder is represented as an icon with the basename bellow

The component is configured with a filter (on entities and file(s) metadata), and also provide a search-box to filter at runtime.

Briefcase widget¶

A widget, using the Briefcase component, SHOULD be available to let the user customize the filter used by the component.

Briefcase view¶

A view containing the widget with no particular filter SHOULD be available in order to offer a global view on all associated file(s).

Backend API¶

The backend MUST provide :

  • a briefcase manager
  • a webservice

Manager¶

The manager MUST provide the following API :

  • an add method, receiving a file and an entity to associate the file to

  • a fetch method, filterable by entities

  • an update method :
    • to rename/move the file in the entity’s folder
    • to move a file from an entity to another

  • a remove method, receiving a filter which will remove all matching files

Webservice¶

The webservice will provide a single route with distinct verbs for each method of the manager.
